# xSmartKG 用户手册

## 功能介绍

xSmartKG 是一个智能知识图谱构建工具，可以从多种格式的文档中自动提取实体和关系，构建知识图谱。

### 主要特性

- 🔄 **多格式支持**: TXT, MD, DOCX, PDF, Excel
- 🤖 **AI 自动提取**: 使用 LLM 智能识别实体和关系
- 📊 **Excel 列映射**: 直接从表格列配置三元组
- ⚡ **实时进度**: WebSocket 实时推送构建进度
- 🗂️ **图空间管理**: 支持多个独立的知识图谱空间

---

## 快速开始

### 步骤 1: 访问上传页面

访问 `/construct/knowledge/upload` 进入知识图谱构建页面。

### 步骤 2: 选择或创建图空间

在左侧面板选择已有图空间，或在中间配置区输入新的图空间名称。

### 步骤 3: 选择处理模式

| 模式 | 适用场景 |
|------|----------|
| 🤖 Auto (LLM) | 非结构化文本 (PDF/Word/TXT) |
| 📊 Column Mapping | 结构化 Excel 表格 |

### 步骤 4: 上传文件

拖拽或点击上传区域选择文件。

### 步骤 5: 开始构建

点击 "Start Building Knowledge Graph" 按钮开始处理。

## Excel 处理模式

### Auto 模式

系统会将 Excel 内容转换为描述性文本，然后使用 LLM 提取实体和关系。

**适用于**: 包含自然语言描述的表格

### Column Mapping 模式

直接配置 Excel 列到三元组的映射：

| 配置项 | 说明 | 示例 |
|--------|------|------|
| Subject Column | 主体实体列名 | "公司名称" |
| Predicate | 关系类型 | "负责人是" |
| Object Column | 客体实体列名 | "负责人" |

**适用于**: 结构化的实体关系表格

---

## Prompt 模板

系统内置 5 个提取模板：

| 模板 | 用途 |
|------|------|
| 📝 默认 | 通用实体关系提取 |
| 👤 人物关系 | 人名、职位、人际关系 |
| 🏢 企业关系 | 公司、投资、合作关系 |
| 📚 知识概念 | 术语、概念层级关系 |
| 📅 事件提取 | 事件、时间、因果关系 |

## 常见问题

### Q: 支持多大的文件？

A: 建议单个文件不超过 50MB，批量上传建议不超过 10 个文件。

### Q: 任务失败怎么办？

A: 点击历史任务查看错误信息，可使用 "Reprocess" 功能重新处理。

### Q: 如何查看构建的图谱？

A: 任务完成后点击 "View Graph" 按钮跳转到图谱可视化页面。
